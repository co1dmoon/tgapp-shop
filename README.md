# MaxxPC Telegram Бот

Телеграм-бот для магазина компьютерной техники MaxxPC.

## Установка

```bash
# Клонирование репозитория
git clone https://github.com/yourusername/maxxpc-tg-bot.git
cd maxxpc-tg-bot

# Установка зависимостей
npm install

# Создание файла .env
cp .env-example .env
# Отредактируйте файл .env, добавив токен вашего бота
```

## Настройка

Отредактируйте файл `.env` и добавьте следующие переменные:

```
BOT_TOKEN=ваш_токен_бота
WEBAPP_URL=http://ваш_домен/catalog.html
```

## Запуск

### Режим разработки

```bash
npm run dev
```

### Запуск только бота

```bash
npm run bot
```

### Боевой запуск

```bash
npm start
```

## Функциональность

- Просмотр каталога товаров
- Добавление товаров в корзину
- Оформление заказа

## Технологии

- Node.js
- Express
- Telegraf (Telegram Bot API)
- Telegram Web App

## Функции

- Просмотр каталога компьютеров
- Добавление товаров в корзину
- Оформление заказа через Telegram Web App
- Админ-панель для управления каталогом и заказами

## Структура проекта

- `index.js` - основной файл приложения
- `services/telegramBot.js` - логика Telegram бота
- `controllers/` - контроллеры для работы с БД
- `routes/` - API маршруты (клиентские и админские)
- `prisma/` - схема и миграции Prisma
- `public/` - статические файлы для веб-приложения
  - `catalog.html` - страница каталога товаров
  - `images/` - изображения товаров (включая `default.svg`)

## Настройка для производства

Для запуска в режиме производства, вам необходимо:

1. Получить домен и SSL-сертификат для веб-приложения
2. Настроить HTTPS (в продакшен-сборке используется Caddy и Let's Encrypt, см. README_DEPLOY.md)
3. Добавить HTTPS URL вашего веб-приложения в настройках бота через BotFather и в `.env`
4. Настроить остальные переменные окружения

## Примечания по разработке

- Для тестирования Web App в Telegram используйте ngrok для получения HTTPS URL
- Админские команды в боте доступны по команде `/admin`
